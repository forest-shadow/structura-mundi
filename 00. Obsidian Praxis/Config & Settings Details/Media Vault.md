# Подход 1 — **Выделить Media-vault**

Это самое простое и безопасное решение:
* Создаёшь отдельный vault, где хранятся картинки/видео/аудио  
* В основном vault — только текст  
* Подключаешь через относительные ссылки или абсолютный путь

**Плюсы:**
- уменьшается размер Git-репозитория (~600 МБ упадёт до десятков)
- ускоряется синхронизация
- резервные копии становятся быстрее

**Это лучший шаг прямо сейчас.**

# Имплементация

В Obsidian **нет встроенной функции “общей медиа-библиотеки”**, но есть **три рабочих, надёжных способа**, которые реально используются в больших хранилищах. Все они позволяют нескольким vault'ам пользоваться одной и той же папкой с медиа **без дублирования** и без “ломания” ссылок.

Даю все способы по убыванию удобства.

---

# **Способ 1 — Симлинк (лучший вариант)**

Это самый простой и надёжный способ создать _единую общую папку_ для картинок, аудио, PDF и любого бинарного контента.

## ❗Работает идеально на Windows, Linux и macOS

И Obsidian воспринимает такую папку абсолютно нормально.

---

## **Как сделать (Windows)**

Можно создать либо **символическую ссылку**, либо **жёсткую ссылку на каталог** (directory junction).

### Вариант 1 — Directory Junction (рекомендуется на Windows)

```powershell
mklink /J "D:\Obsidian\MindTrust\_media" "D:\Obsidian\MediaVault"
```

### Вариант 2 — Симлинк

```powershell
mklink /D "D:\Obsidian\MindTrust\_media" "D:\Obsidian\MediaVault"
```

---

## **Как сделать (Linux / WSL / macOS)**

```bash
ln -s /home/user/Obsidian/MediaVault /home/user/Obsidian/MindTrust/_media
```

## Что даёт такой подход

- **Физически медиа живут в одном месте**
- В каждом vault они выглядят как локальная папка
- Внутри Obsidian можно вставлять изображения обычным способом
- Git в главном vault _не трекает бинарные файлы_ (если `.gitignore` прописан правильно)

## Плюсы

- Самый удобный вариант
- Не требует изменения привычного workflow
- Изображения/файлы доступны в autocomplete при вставке
- Работает даже если подключены несколько vault'ов одновременно
- Ничего не ломает в ссылках

## Минусы

- На мобильном Obsidian нет симлинков → если ты используешь телефон для работы с медиа, нужно продумать схему синхронизации
    

# **Способ 2 — Внешняя папка + относительные ссылки**

Вариант, который любят пользователи, использующие Git или rsync.
### Суть:

- Media хранятся в отдельной директории **вне всех vault'ов**
- В заметках используются **относительные ссылки вида:**  
    `![[../../MediaVault/images/photo01.png]]`

### Пример структуры:

```
/Obsidian/
  /MindTrust/
  /Projects/
  /Daily/
  /MediaVault/   ← общая медиа-библиотека
```

Ты просто вставляешь файл в заметку через путь.

## Плюсы

- Чистый Git-репозиторий
- Никаких проблем с множеством медиа
- Работает на всех устройствах

## Минусы

- Относительные ссылки иногда длинные
- Перемещать файлы нужно вручную

# **Что лучше выбрать лично тебе?**

Учитывая твой стиль (ты активно работаешь на Windows + WSL, используешь Git, много конспектов, много медиа):

💡 **Я рекомендую Способ 1 — SymLink / Junction**

Потому что
- он работает как _прозрачный alias_ в файловой системе
- Obsidian считает папку локальной и показывает медиа в preview
- Git можно настроить так, чтобы медиа не коммитились
- всё работает быстро
- нет хрупких относительных путей
