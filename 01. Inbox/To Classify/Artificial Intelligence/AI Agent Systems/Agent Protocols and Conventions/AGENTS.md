# AGENTS.md

## Краткое определение

`AGENTS.md` — это репозиторно-локальный файл инструкций для кодовых агентов, в котором фиксируются ожидания проекта, рабочие соглашения, команды проверки, архитектурные ограничения и локальные нормы поведения.

## Основная идея или механизм

В практике Codex `AGENTS.md` выступает машинно-читабельной формой инженерной памяти проекта. Он сокращает стоимость повторного вхождения агента в репозиторий и переводит неявные командные договорённости в явный операционный контекст.

Иерархически `AGENTS.md` может существовать как глобальный файл в домашней директории Codex и как файл в корне репозитория. Для более узких областей допускаются локальные override-файлы. Это делает `AGENTS.md` слоем scoped-инструкций, а не просто заметкой о проекте.

## Границы темы

- Входит: команды тестирования, соглашения по структуре кода, правила документации, ограничения по зависимостям, локальные инженерные инварианты.
- Не входит: общая пользовательская документация проекта. Эту роль обычно выполняет `README`.
- Не входит: нормативное решение о допуске команды к исполнению. Эту роль выполняют `[[Codex Rules]]`.

## Практические применения

- В программировании: фиксация того, какие проверки запускать, где лежит доменная логика, какие файлы нельзя трогать без согласования.
- В управлении проектами: кодирование workflow-команд, критериев готовности, процедур review и branch discipline.
- В личной эффективности: создание персональной рабочей памяти для собственных проектов, шаблонов задач и повторяемых инженерных ритуалов.

## Типичные ошибки

- Писать в `AGENTS.md` пользовательский маркетинговый текст вместо операционных инструкций.
- Дублировать в нём всё содержание репозитория без отбора по агентной полезности.
- Смешивать проектную память с жёсткими правилами безопасности и политики исполнения.
