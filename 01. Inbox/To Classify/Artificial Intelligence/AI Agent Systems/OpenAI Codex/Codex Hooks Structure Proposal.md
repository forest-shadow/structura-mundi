# Codex Hooks Structure Proposal

## Краткое определение

`Codex Hooks` разумно держать как дочернюю `article`-заметку внутри `OpenAI Codex`.

## Рекомендуемая роль в иерархии

```text
OpenAI Codex
└── Codex Hooks
```

## Почему так

- Это частный механизм расширения и автоматического реагирования внутри конкретной среды.
- Тема концептуально соседствует с plugins и skills.
- Отдельная более глубокая ветка пока не нужна.

## Что не стоит создавать сейчас

- отдельные дочерние статьи под каждый event hook без корпуса заметок;
- переносить hooks в общую протокольную ветку, где они потеряют tool-specific контекст.
