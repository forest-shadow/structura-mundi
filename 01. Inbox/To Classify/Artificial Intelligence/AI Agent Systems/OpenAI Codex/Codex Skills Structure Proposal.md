# Codex Skills Structure Proposal

## Краткое определение

`Codex Skills` разумно держать как дочернюю `article`-заметку внутри `OpenAI Codex`.

## Рекомендуемая роль в иерархии

```text
OpenAI Codex
└── Codex Skills
```

## Почему так

- Это частный механизм специализации агента внутри конкретной платформы.
- Тема тесно связана с plugins и локальными instructions, но не совпадает с ними.
- Пока не требуется отдельный overview про skill ecosystem.

## Что не стоит создавать сейчас

- отдельные дочерние статьи под каждый вид skill;
- переносить skills в общую протокольную ветку, если речь идет о Codex-specific механике.
