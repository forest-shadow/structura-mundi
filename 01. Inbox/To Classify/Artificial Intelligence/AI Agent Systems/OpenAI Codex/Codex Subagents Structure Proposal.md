# Codex Subagents Structure Proposal

## Краткое определение

`Codex Subagents` разумно держать как дочернюю `article`-заметку внутри `OpenAI Codex`.

## Рекомендуемая роль в иерархии

```text
OpenAI Codex
└── Codex Subagents
```

## Почему так

- Это важный, но все еще tool-specific механизм декомпозиции и orchestration.
- Тема полезна как самостоятельная статья, но пока не требует собственного поддерева.
- Общую тему multi-agent systems можно добавить позже как соседнюю ветку, а не как вложение только в Codex.

## Что не стоит создавать сейчас

- отдельный overview только под explorers/workers без соседних статей;
- смешивать общую многоагентность как дисциплину с частной Codex-реализацией.
