# AI-Assisted Programming Structure Proposal

## Краткое определение

`AI-Assisted Programming` на текущем этапе разумно держать как дочернюю `article`-заметку внутри `AI-Assisted Software Development`.

## Рекомендуемая роль в иерархии

```text
AI-Assisted Software Development
└── AI-Assisted Programming
```

## Почему так

- Это уже отдельный устойчивый аспект software engineering.
- Тема пока не требует собственного поддерева.
- Инфраструктурные механизмы лучше переиспользовать через `related`, а не переносить внутрь этой ветки.

## Что не стоит создавать сейчас

- отдельный overview только под code generation;
- отдельные дочерние ветки под `debugging`, `testing` и `review`, пока они не оформлены как самостоятельные заметки.
