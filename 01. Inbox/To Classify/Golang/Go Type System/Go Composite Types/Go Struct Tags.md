# Go Struct Tags

## Определение

`Go Struct Tags` - это строковые метаданные, прикреплённые к полям `struct` в объявлении типа.

## Почему это отдельная тема

Теги не просто сопровождают `struct` как случайная деталь синтаксиса. Это устойчивый языковой механизм, через который библиотеки и фреймворки получают договорённости о сериализации, валидации, маппинге и других формах интерпретации полей.

## Что здесь стоит рассматривать

- синтаксис тегов в объявлении поля структуры;
- различие между именем поля, типом поля и его тегом;
- сырая строка тега и соглашения вида `json:"name,omitempty"`;
- роль reflection при чтении тегов во время выполнения;
- границы между языковым механизмом и библиотечными конвенциями.

## Что важно не смешивать

`Go Struct Tags` - это не то же самое, что `Go Reflection`, хотя на практике теги чаще всего читаются именно через reflection. Языковая форма тега относится к устройству `struct`, а механика извлечения значения тега относится к runtime-inspection слою.

## Минимальные примеры

```go
type User struct {
    Name string `json:"name"`
    Age  int    `json:"age,omitempty"`
}
```

В этом примере теги не меняют сам тип `User`, но дают внешним механизмам данные о том, как интерпретировать поля.
