# Page Fault

## Краткое определение

`Page Fault` — это событие, при котором обращение к виртуальной странице не может быть завершено по быстрому пути и требует вмешательства операционной системы.

## Почему тема находится в ветке Paging

Эта заметка относится к paging-механике, потому что page fault описывает поведение системы в момент, когда нужная страница отсутствует, недоступна или требует дополнительной обработки.

## Основная идея или механизм

- процесс обращается по виртуальному адресу;
- аппаратная часть или page table state показывают, что страница не готова к обычному доступу;
- управление передается ядру, которое решает, можно ли подгрузить страницу, обновить отображение или завершить процесс ошибкой.

## Границы темы

- Входит: page fault как событие и часть control flow virtual memory.
- Не входит: полный разбор swap policy, page replacement algorithms и всех видов memory exceptions.

## Примеры, случаи или следствия

Эта заметка должна показать, почему page fault не всегда означает ошибку программы: он может быть и нормальной частью demand-paging поведения.
