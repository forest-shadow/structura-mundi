# Virtual Address Space

## Краткое определение

`Virtual Address Space` — это логическое пространство адресов, которое видит процесс, независимо от реального расположения данных в физической памяти.

## Почему тема находится в ветке Virtual Memory

Эта заметка нужна, чтобы зафиксировать саму абстракцию, ради которой существует virtual memory. Без нее механизмы `Paging`, `Page Table` и `TLB` превращаются в набор низкоуровневых деталей без понятной цели.

## Основная идея или механизм

- каждый процесс работает с собственным набором виртуальных адресов;
- эти адреса не обязаны напрямую совпадать с физическими;
- операционная система и аппаратная поддержка переводят их в реальные размещения данных.

## Границы темы

- Входит: абстракция адресного пространства процесса и ее роль в изоляции и организации памяти.
- Не входит: полный разбор формата page tables, TLB caching и всех вариантов page replacement.

## Примеры, случаи или следствия

Эта заметка должна показать, почему процесс может воспринимать память как непрерывное личное пространство даже тогда, когда данные физически разбросаны по разным областям RAM и вторичного хранения.
