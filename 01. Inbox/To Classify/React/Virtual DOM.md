# Virtual DOM

## Краткое определение

Каноническая статья про `Virtual DOM` как внутреннее представление дерева интерфейса, которое помогает объяснять, как React вычисляет изменения до применения их к реальному DOM.

## Основная идея или механизм

Эта заметка должна объяснять, зачем React использует промежуточное представление UI, как оно связано с render и reconciliation и почему `Virtual DOM` полезно понимать как объясняющую абстракцию, а не как полное описание всей современной rendering-модели React.

## Границы темы

- Входит: идея промежуточного представления UI, связь с reconciliation и сравнением деревьев.
- Не входит: вся тема планирования обновлений, hooks или общая архитектура React целиком.
