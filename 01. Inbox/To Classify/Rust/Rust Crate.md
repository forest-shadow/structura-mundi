# Rust Crate

## Краткое определение

`Rust Crate` — это базовая единица компиляции и организации кода в Rust, из которой строятся библиотеки и исполняемые программы.

## Основная идея или механизм

Понятие crate задает границу сборки, видимости и публикации кода на уровне языка и toolchain. Через него Rust организует, какой код компилируется вместе, как он экспортирует API и как другие части программы или экосистемы на него ссылаются.

## Границы темы

- В эту тему входит: crate как единица компиляции, различие между library crate и binary crate, его роль в организации проекта.
- Рядом, но не тождественно ей: package в Cargo, module, workspace, visibility и dependency graph.

## Примеры, случаи или следствия

- Один package может содержать один library crate и один или несколько binary crates.
- Публичная поверхность crate определяет, какие типы и traits становятся внешним API.
- Границы crate влияют на то, как проект делится на переиспользуемые и компилируемые части.
