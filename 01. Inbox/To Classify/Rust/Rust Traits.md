# Rust Traits

## Краткое определение

`Rust Traits` — это механизм задания общего поведения, который описывает набор методов и связанных контрактов, которые тип может реализовать.

## Основная идея или механизм

Traits в Rust играют роль интерфейсоподобной абстракции, но не сводятся к простому аналогу interfaces из других языков. Через них выражаются ограничения для generics, полиморфизм по поведению и часть архитектурных границ между типами и библиотечными API.

## Границы темы

- В эту тему входит: что такое trait как языковая сущность, зачем он нужен, как задает поведенческий контракт и как связан с реализацией для типов.
- Рядом, но не тождественно ей: trait bounds, trait objects, associated types, blanket implementations и динамическая диспетчеризация.

## Примеры, случаи или следствия

- Один и тот же trait может быть реализован несколькими типами, если они поддерживают одно и то же поведение.
- Generic-функция может требовать от параметра реализацию определенного trait.
- Публичный API crate часто выражает свои расширяемые контракты именно через traits.
