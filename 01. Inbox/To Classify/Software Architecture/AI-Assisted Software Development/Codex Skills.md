# Codex Skills

## Краткое определение

`Codex Skills` — это модульные самодостаточные единицы специализации агента, которые задают, когда и как Codex должен выполнять определённый тип работы, и при необходимости включают инструкции, скрипты, справочные материалы и дополнительные ресурсы.

## Основная идея или механизм

Базовой единицей навыка является директория с файлом `SKILL.md`, содержащим metadata и инструкции. При необходимости навык дополняется `scripts`, `references`, `assets` и UI-метаданными. Тем самым `Skill` представляет собой не просто prompt, а управляемый пакет процедурного знания.

С точки зрения архитектуры навыки существуют для того, чтобы сделать поведение агента более предметным и повторяемым без перегрузки основного системного контекста. Они реализуют принцип progressive disclosure: базовая метаинформация всегда короткая, а детальный материал подгружается по мере необходимости.

## Границы темы

- Входит: trigger description, workflow instructions, встроенные скрипты, доменные reference-материалы.
- Не входит: упаковка пользовательского расширения в целом. Это относится к `[[Codex Plugins]]`.
- Не входит: глобальные или репозиторные рабочие соглашения. Это относится к `[[AGENTS.md]]`.

## Практические применения

- В программировании: миграции, code review, документация, работа с конкретными API, генерация инфраструктурных артефактов.
- В управлении проектами: triage, release-checklists, подготовка статусов, синхронизация артефактов и проектной документации.
- В личной эффективности: персональные workflow для заметок, исследований, повторяющихся отчётов, почтовых и календарных сценариев.

## Типичные ошибки

- Делать `SKILL.md` слишком длинным и размытым.
- Смешивать в одном навыке несколько слабо связанных процессов.
- Пытаться заменить навыками весь policy- и architecture-layer проекта.
