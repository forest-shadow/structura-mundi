# Single Responsibility Principle

## Краткое определение

Каноническая статья про принцип, согласно которому модуль должен иметь одну основную причину для изменения, а не смешивать несколько независимых обязанностей.

## Основная идея или механизм

`SRP` уменьшает связность изменений и делает код проще для понимания, тестирования и эволюции.

## Границы темы

- Входит: обязанности модуля и причины изменения.
- Не входит: полный разбор всех принципов SOLID.
