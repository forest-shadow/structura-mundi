# UML Behavioral Diagrams

## Краткое определение области

`UML Behavioral Diagrams` - это обзорная ветка для диаграмм, которые показывают, как система ведет себя во времени: какие сценарии поддерживает, как текут действия и как меняются состояния.

## Что входит в эту тему

- `[[UML Use Case Diagram]]`
- `[[UML Sequence Diagram]]`
- `[[UML Activity Diagram]]`
- `[[UML State Machine Diagram]]`

## Как устроена ветка

- `UML Use Case Diagram` удерживает внешний взгляд на цели акторов и границы системы;
- `UML Sequence Diagram` показывает порядок сообщений между участниками сценария;
- `UML Activity Diagram` описывает поток действий и развилки;
- `UML State Machine Diagram` фиксирует возможные состояния сущности и события переходов.

## Рекомендуемый маршрут чтения

1. Начать с `[[UML Use Case Diagram]]`.
2. Затем перейти к `[[UML Sequence Diagram]]`.
3. После этого читать `[[UML Activity Diagram]]`.
4. Завершить `[[UML State Machine Diagram]]`.
