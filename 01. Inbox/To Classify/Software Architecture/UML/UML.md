# UML

## Краткое определение области

`UML` - это обзорный узел для языка визуального моделирования, через который описывают структуру, поведение и развертывание программной системы.

## Что входит в эту тему

- `[[UML Structural Diagrams]]`
- `[[UML Behavioral Diagrams]]`

## Как устроена ветка

- `UML Structural Diagrams` удерживает статическую сторону системы: типы, компоненты и размещение частей;
- `UML Behavioral Diagrams` удерживает сценарии, потоки действий и смену состояний;
- сама заметка `UML` нужна как общий вход: что это за язык, где его границы и зачем разделять разные типы диаграмм, а не смешивать их в одну картинку.

## Рекомендуемый маршрут чтения

1. Начать с общей идеи `UML`.
2. Затем перейти к `[[UML Structural Diagrams]]` и `[[UML Class Diagram]]`.
3. После этого читать `[[UML Component Diagram]]` и `[[UML Deployment Diagram]]`.
4. Затем перейти к `[[UML Behavioral Diagrams]]`, `[[UML Use Case Diagram]]` и `[[UML Sequence Diagram]]`.
5. Завершить `[[UML Activity Diagram]]` и `[[UML State Machine Diagram]]`.
