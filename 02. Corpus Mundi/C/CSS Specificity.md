# 1. CSS Specificity

Специфичность (CSS Specificity) представляет собой формальный механизм лексикографического сравнения селекторов, применяемый для разрешения конфликтов между конкурирующими декларациями одного уровня каскада. Она определяет относительный вес селектора и является критическим фактором предсказуемости стилистической системы.

> <font color="#fb4934">CSS Specificity</font> — это формализованный алгоритм вычисления веса селектора, используемый при возникновении конфликта нескольких деклараций, претендующих на определение значения одного и того же свойства одного и того же элемента.

В иерархии `[[CSS Cascade]]` специфичность не является универсальным принципом выбора правила. Она функционирует как локальный механизм взвешивания исключительно в тех случаях, когда:

- Несколько правил применимы к конкретному элементу.
- Правила содержат конфликтующие декларации идентичных CSS-свойств.
- Конфликт не был разрешен на более приоритетных этапах (Origin, Importance или Cascade Layers).

# 2. Этимология и происхождение термина

Термин восходит к английскому _specific_ («конкретный», «точно определяющий»). В контексте каскада это обозначает степень структурной определённости селектора: чем уже и точнее селектор описывает целевой элемент, тем выше его специфичность и, следовательно, относительный приоритет.

# 3. Историческое развитие и эволюция концепта

Механизм специфичности присутствует в CSS с ранних этапов стандартизации как базовое средство разрешения конфликтов.

**Эволюция подхода:**
- **Ранний этап:** Специфичность часто преподносилась как доминирующий принцип каскада. Данная дидактическая модель была удобна для обучения, но концептуально неполна.
- **Современный этап:** Специфичность строго локализована внутри общей модели каскада. Она рассматривается как один из уровней разрешения противоречий наряду с происхождением правил (Origins), важностью (`!important`), слоями (`@layer`) и порядком следования (Source Order). Уточнение её места в спецификации позволило создать более предсказуемую инженерную среду.

# 4. Формальные свойства и структура

Специфичность определяется исключительно структурным составом селектора, а не его семантикой или визуальной значимостью.

## Ключевые инварианты:

- Специфичность не зависит от значения или имени свойства.
- Специфичность не зависит от положения элемента в DOM-дереве, если это положение не отражено в самом селекторе.
- Специфичность не зависит от того, насколько визуально заметно изменение.

## Компоненты веса:

Классификация компонентов селектора для вычисления веса включает три основные категории:
1. **ID-селекторы:** Обладают наивысшим приоритетом в рамках специфичности (например, `#main`).
2. **Селекторы классов, атрибутов и псевдоклассов:** Средний уровень приоритета (например, `.nav`, `[type="text"]`, `:hover`).
3. **Селекторы типов/тэгов и псевдоэлементов:** Низший уровень приоритета (например, `div`, `h1`, `::before`).

Универсальный селектор (`*`), комбинаторы (`+`, `>`, `~`, ) и служебные конструкции (например, `:where()`) не вносят вклада в вес селектора.

# 5. Математическое ядро (Формальная модель)

Специфичность представляется как упорядоченный кортеж (вектор) из трех разрядов: **(A, B, C)**.

## Формула вычисления:

- **A**: Количество ID-селекторов.
- **B**: Количество селекторов классов, атрибутов и псевдоклассов.
- **C**: Количество селекторов типов и псевдоэлементов.

## Лексикографический порядок сравнения:

Сравнение векторов выполняется строго поразрядно слева направо:
- Сначала сравнивается разряд **A**. Если значения различаются, селектор с большим значением признается более специфичным.
- При равенстве в разряде **A** сравнивается разряд **B**.
- При равенстве в разрядах **A** и **B** сравнивается разряд **C**.

**Важное следствие:** Никакое количество компонентов младшего разряда не может перевесить наличие одного компонента в старшем разряде. Один ID всегда «сильнее» бесконечного количества классов.

## Функциональные псевдоклассы:

- `:is()`, `:not()`, `:has()`: Вклад в специфичность равен вкладу самого «тяжелого» селектора, переданного в качестве аргумента.
- `:where()`: Специально спроектирован для обладания нулевой специфичностью (0, 0, 0) независимо от сложности аргументов.

# 6. Современное значение и архитектурная интерпретация

В современной разработке специфичность — это не только техническая формула, но и индикатор качества архитектуры CSS.

**Инженерные аспекты:**
- **Управляемость:** Низкая и предсказуемая специфичность позволяет легко переопределять стили без использования «грязных» хаков.
- **Масштабируемость:** Чрезмерно высокая специфичность ведет к «войне селекторов» и хрупкости системы, когда каждое новое исправление требует всё более тяжелых конструкций.
- **Дисциплина проектирования:** Использование методологий (например, BEM) направлено на удержание специфичности на минимально необходимом уровне.

# 7. Сравнение с соседними механизмами каскада

Для корректной работы необходимо четко разграничивать специфичность и другие факторы:
- **Specificity vs Cascade:** Каскад — это вся система приоритетов; специфичность — лишь одна из ступеней внутри этой системы.
- **Specificity vs Source Order:** Порядок следования вступает в силу только при полной идентичности векторов (A, B, C) на одном уровне каскада.
- **Specificity vs !important:** `!important` не увеличивает специфичность, а переводит декларацию на другой уровень важности (Importance), который выше уровня специфичности.
- **Specificity vs @layer:** Слой каскада определяет глобальный контекст. Правило из более приоритетного слоя победит правило из менее приоритетного слоя, даже если у последнего специфичность выше.

# 8. Примеры и случаи сравнения

Ниже приведены примеры лексикографического расчета веса (A, B, C).

**Простые случаи:**
- `button` → (0, 0, 1)
- `.btn` → (0, 1, 0) — _побеждает `button`_.
- `main article p` → (0, 0, 3)
- `.text` → (0, 1, 0) — _побеждает `main article p`, так как разряд классов старше разряда типов_.

**Преимущество ID:**
- `.page .content .article p` → (0, 3, 1)
- `#main` → (1, 0, 0) — _побеждает, так как ID находится в самом старшем разряде_.

**Равная специфичность (Source Order):**
- `.card .title { color: black; }` → (0, 2, 0)
- `.panel .title { color: blue; }` → (0, 2, 0)
    
- _Результат зависит от того, какое правило написано в CSS-файле позже._

**Случай проигрыша более специфичного правила:**
- `#app .button { color: black; }` → (1, 1, 0)
- `.button { color: blue !important; }` → (0, 1, 0)
    
- _Несмотря на колоссальную разницу в специфичности, победит `blue !important`, так как механизм Importance приоритетнее._

# 9. Типичные ошибки, заблуждения и ограничения

- **Суммирование как десятичных чисел:** Ошибочно полагать, что 10 тегов заменяют 1 класс. Это векторы, а не числа.
- **Длина селектора:** Визуально длинный селектор не обязательно является более специфичным. Важен только компонентный состав.
- **Эскалация специфичности:** Попытка решить любой конфликт путем добавления ID или вложенности ведет к деградации кодовой базы.
- **Интуитивное вычисление:** Современные псевдоклассы (`:is()`, `:has()`) делают расчет менее очевидным. При сомнениях следует обращаться к формальной спецификации.

# 10. Краткий вывод

CSS Specificity — это точный инструмент взвешивания селекторов, работающий «в тени» более крупных механизмов каскада (Origins, Layers). Его основная задача — разрешение локальных конфликтов правил. Для создания устойчивых систем следует стремиться к минимально возможной и однородной специфичности во всей кодовой базе.
